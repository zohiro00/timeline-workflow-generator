# AGENTS.md

このリポジトリの開発憲法です。ここに書かれた原則は、詳細な細則より優先します。
細則は [docs/development/](docs/development/) 配下に分けて管理し、リンク先もこのファイルと同じ効力を持ちます。

## 憲法

- 実際の環境、実行結果、コード、設定とドキュメントを一致させます。不一致を見つけたら、コードか文書のどちらかを直して一致させてください。
- 変更に強い設計を優先します。DSL は宣言的に保ち、ユーザーに座標、空白、ハイフン数、描画位置の微調整を要求しません。
- テストなしの機能変更、バグ修正、リファクタリングは不可です。
- `main` への直接 push は禁止です。変更は必ず作業ブランチで行い、レビュー可能な差分として提出してください。
- 検証していないことを検証済みとして報告しません。失敗・未実行・未確認は、そのまま明記してください。
- 依存関係の管理とスクリプト実行は `pnpm` に統一します。npm や別のパッケージマネージャーへのフォールバックを追加しません。
- ユーザーの未コミット変更を上書きしません。既存差分がある場合は、内容を確認してから作業してください。
- 変更範囲を狭く保ちます。依頼に無関係なリファクタリング、配色変更、文言変更、依存更新は混ぜないでください。

## 最低限の確認

- 作業前: `git status --short` と現在ブランチを確認してください。
- 変更後: `pnpm test`、`pnpm run guard:change`、`git diff --check` を実行してください。
- UI、SVG 出力、ビルド設定、依存関係に関わる変更後: `pnpm run build` も実行してください。
