# UI と SVG の方針

- UI は生成エンジンを使うための作業画面として扱います。装飾よりも、入力、プレビュー、設定、ダウンロードの分かりやすさを優先します。
- SVG は PowerPoint などの資料へ貼りやすい出力を前提にします。白背景、読みやすい文字、明確なレーン、時系列グリッドを壊さないでください。
- 画面上の説明文で不足を補うより、自然に操作できるコントロールと状態表示を作ってください。
- レスポンシブ対応では、テキストやコントロールが重ならないことを確認してください。
- 日本語のUI文言は、不自然な位置で改行されにくい表示を意識してください。狭い幅で表示されるラベル、通知、ボタン、ステータスでは、必要に応じて `line-break: strict`、`text-wrap: pretty`、`overflow-wrap`、`word-break` などを使い、読みやすさと横あふれ防止の両方を確認してください。
