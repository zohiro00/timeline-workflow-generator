# テスト規約

- テストなしの機能変更、バグ修正、リファクタリングは不可です。
- 既存テストは [../../test/workflow.test.js](../../test/workflow.test.js) にあります。コアロジックを変更する場合は、同じファイルまたは近い粒度のテストを追加してください。
- バグ修正では、最初に失敗するテストを追加し、その後に実装修正を行うことを原則とします。
- パーサー変更では、正常系だけでなく、行番号、重複、未定義ノード、未定義レーン、不正構文などのエラー系も確認してください。
- レイアウト変更では、ルート、分岐、合流、複数レーン、循環検出を壊していないことをテストしてください。
- レンダリング変更では、完全な SVG 全体のスナップショットに頼りすぎず、重要な属性、クラス、座標、エスケープ結果をピンポイントで検証してください。
- ドキュメントのみの変更でも、作業後に `pnpm test` を実行し、結果を報告してください。実行できない場合は理由を明記し、代替コマンドで済ませないでください。
